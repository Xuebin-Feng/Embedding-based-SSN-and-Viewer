Sequence Similarity Network (SSN) Viewer

Command Line Interface (CLI) Manual & Reference Guide

This reference documents all available visual, physical, and community-clustering commands. Use the sidebar to search or filter commands dynamically. Core commands operate on the visual state in memory, while external scripts execute advanced bioinformatics algorithms and export outputs to the disk.

Expression Interpreter Syntax Rules

Overview: The Common Expression Interpreter is the underlying engine that parses text queries into mathematical selections. It is shared across multiple tools in the SSN software (like color, hide, label, etc.) and allows for highly complex, multi-layered targeting of network nodes.

⚠️ Important Space Rule

Expressions must be written continuously without spaces. Spaces are strictly reserved to separate different commands, parameters, or attributes.

Targeting Modules

The interpreter recognizes six distinct targeting syntaxes:

1. Text & Wildcard Matching ("Text")

Surrounding text in double quotes searches node FASTA headers.

  • Exact Substring: "kinase" matches any header containing the word kinase.
  • Wildcards: Supports standard glob patterns. "*A5?T*" matches any header containing 'A5', any single character, and 'T'.
2. Attribute Labels (#Label#)

Surrounding text in hash symbols searches internal network metadata, clusters, and manually assigned groups.

  • Clusters: #cluster_1#, #cluster_2#, etc.
  • Noise: #noise# isolates unclustered/filtered nodes (Cluster -1).
  • Custom Groups: #human# matches nodes you have manually grouped and labeled as "human".
3. Sequence Alignment Queries ([AA][Pos])

Allows targeting based on the Multiple Sequence Alignment (MSA) residue distribution.

  • Specific Amino Acid: P106 targets all nodes that have a Proline (P) at alignment column 106.
  • Gap Queries: _100 targets all nodes that have a gap (- or .) at alignment column 100.
4. External File Cross-Referencing (@Filename@)

Surrounding a filename with the @ symbol cross-references the network against a text file or FASTA file located in the Header_Lists directory.

  • Direct Match: @my_targets.txt@ checks if the network header matches lines in the file exactly.
  • NCBI Extraction: @[NCBI]my_targets.txt@ automatically scans the text file for standard NCBI accession codes (e.g., WP_01234.1) and matches the network headers containing those codes, ignoring surrounding text.
  • PDB Extraction: @[PDB]my_targets.txt@ performs the same extraction but for Protein Data Bank codes (e.g., 1XYZ).
5. UI Selection ($sele$)

Typing $sele$ explicitly passes the nodes you currently have highlighted/selected with your mouse in the viewer.

6. Metadata Queries ({Key Op Val})

Allows targeting based on custom metadata fields uploaded using Excel or CSV tables.

  • Numerical: {Length>500} targets nodes where sequence length is greater than 500.
  • Text: {Organism=*coli*} targets nodes where Organism metadata contains "coli".
Boolean Logic Operators

You can combine any of the targeting modules above using standard Boolean logic (no spaces allowed):

  • & (AND): "Human"&#cluster_1# (Must be human AND in cluster 1).
  • | (OR): P106|A106 (Must have either Proline OR Alanine at position 106).
  • ! (NOT): !#noise# (Must NOT be noise).
  • ^ (XOR): #cluster_1#^P106 (Exclusive OR - either in cluster 1 or has Proline at 106, but not both).
  • () (Grouping): Parentheses can be used to enforce order of operations, e.g., !(#cluster_1#|#cluster_2#).
Core Commands
zoom – Precise Camera Control
zoom <width>

Function: Set the camera view width to exactly the specified numeric <width>+10% while keeping the current center point of the view intact

Examples
  • zoom 500
save – Save Current View to Cache
save [filename / filename.h5]

Function: Saves the current view to an HDF5 cache file. The cache file is associated with the current layout. If no filename is provided, it automatically generates an incrementing version number (e.g., _ver.01.h5, _ver.02.h5) in the target layout folder to prevent overwriting previous saves.

Examples
  • save
  • save session_layout.h5
reset – Reset Selected View to Initial States
reset <target_1> [target_2] ... or <target_1> [target_2] ... reset

Function: Reset selected properties to their initial states. Supported targets include:

Supported Targets
  • color / colors: Reverts all node colors to the default neighbor color.
  • size / sizes: Resets all node sizes to the baseline size defined in configuration.
  • shape / shapes: Resets all nodes back to standard circular 'disc' shapes.
  • cluster / clusters: Strips all cluster labels, removes cluster UI visuals, and clears the cluster tooltip data.
  • group / groups: Clears all custom group label sets assigned to nodes.
  • hide / hidden: Unhides all hidden nodes by setting the entire visibility mask back to True.
  • network / networks: Snaps all node positions back to the original starting layout coordinates (or the state loaded from the last .h5 save).
Examples
  • reset colors sizes
  • reset network hide
hide – Hide Selected Nodes from View
hide [expression]

Function: hide the currently selected nodes and any edges connecting to them. The number of currently hidden nodes are shown at the bottom right of the screen. Note that the “hide” command does not really change the layout, meaning that it will not exclude hidden nodes from analyses.

Arguments & Modes
  • Without arguments: Immediately hides all currently selected nodes and their connected edges.
  • With single or free: Hides all visible nodes that have no active edges at the current similarity threshold.
  • With expression: Hides all visible nodes matching the logical expression (e.g. hide #noise#).
💡 Unhiding Nodes

To unhide nodes and restore full visibility, use the reset hide command.

Examples
  • hide
  • hide single
  • hide #noise#
undo & redo – Session History Navigation
undo | redo

Function: redo and undo last actions. Most commands automatically save visual and physical viewer states to history prior to execution, allowing for instantaneous revert and reload.

Examples
  • undo
  • redo
run – Execute Command Script
New
run [filename]

Function: Opens a file explorer to select a command script file (.txt or .py) and executes the commands in sequence.

Execution Details
  • Python Scripts (.py): Executed in a subprocess using the active python environment. The script's standard output (stdout) is read and executed line-by-line in the viewer.
  • Text Files (.txt): Executed line-by-line. Comments starting with double slashes (//) are automatically ignored.
  • Infinite Loop Protection: Recursive run calls inside a script are automatically ignored to prevent viewer freezes.
Examples
  • run
Alignment
reference – Reset Reference to a Different Sequence
reference [target]

Function: Dynamically changes the active sequence used as the global alignment reference and reloads the multiple sequence alignment to use the new reference indexing. The <target> can be a substring or wildcard that matches a specific FASTA header. If a substring matches multiple headers, it defaults to the first match and issues a warning. Calling reference without an argument will display the current active reference name.

Examples
  • reference
  • reference E1_RA
alignment – Switch Active Alignment (MSA)
New
alignment [alignment_identifier] or alignment list

Function: Dynamically changes the active Multiple Sequence Alignment file (.fasta or .h5) used by the viewer and reloads the MSA mapping.

Arguments
  • Without arguments: Opens a file dialog explorer to manually select a FASTA or H5 MSA file.
  • alignment_identifier: An index number or a case-insensitive substring query matching a file in the Multiple Alignments folder.
  • list: Prints a list of all available alignment files in the Multiple Alignments folder to the console and identifies which one is currently active.
⚠️ Loading Rules & Integrity Checks

The sequence headers currently in the viewer must be a strict subset of the sequence headers present in the new MSA file. If any node sequence is missing, the load fails and the viewer automatically rolls back to the previous alignment to maintain session stability.

Examples
  • alignment
  • alignment list
  • alignment 3
  • alignment kinase_msa
query – Alignment Position Query Tool
query [expression] <positions>

Overview: The query command allows you to instantly interrogate the multiple sequence alignment (MSA) data directly from the viewer. It calculates the amino acid frequencies, consensus distributions, and gap percentages for specific column positions.

📋 Prerequisites

Requires an active MSA loaded and a reference sequence set (via reference command) to anchor position coordinate numbers.

Targeting & Fallbacks
  • Explicit Expression: Runs query on nodes matching the expression (e.g. #cluster_1#).
  • Mouse Selection (Fallback 1): If expression is omitted, queries the nodes currently selected in the viewer.
  • Global Network (Fallback 2): If no expression is provided and no selection exists, queries ALL nodes in the network.
Core Arguments
  • Positions: Specific columns inside brackets, supporting ranges and single positions (e.g., [10-25] or [15,30,45]).
  • Insertions (Decimals): Insertion residues are marked using decimals (e.g., 106.1, 106.2). Querying [106-107] natively prints statistics for 106, 106.1, 106.2, and 107 in order.
Examples
  • query [10, 15, 20-30]
  • query P106 [150-160]
  • query {Length>500} [10-20]
Analysis
cluster – Topology Clustering Tool
cluster [mode] [parameter] [min_size]

Overview: The cluster command partitions the currently loaded sequence similarity network into distinct functional communities. It applies color-coded labels to the nodes and records these boundaries for downstream tools (such as sub-alignment and group extraction).

Arguments
  • MODE: The clustering algorithm to apply: leiden, mcl, or jaccard. Defaults to leiden.
  • PARAMETER: The primary tuning variable for the chosen algorithm (resolution or inflation).
  • MIN_SIZE: An integer defining the minimum number of nodes required to form a valid cluster. Smaller communities are classified as Noise (grey) and ignored by downstream tools. Defaults to 10.
Algorithms
Leiden (Default)

Mechanism: Optimizes network modularity and density. Highly efficient for massive networks and guarantees well-connected communities. Automatically leverages edge similarity scores as weights. Requires leidenalg and igraph.

Parameter: Resolution (Default: 1.0). Increasing this value forces the network to fracture into smaller, more numerous clusters.

Usage: cluster leiden 1.0 10 (or simply cluster 1.0 10)

MCL (Markov Clustering)

Mechanism: Simulates random walks through the network, assuming that a random walker stays within a dense cluster rather than crossing a sparse boundary. Uses similarity scores as transition probabilities. Requires markov_clustering and networkx.

Parameter: Inflation (Default: 2.0). Controls rigidity of boundaries. Higher values (e.g. 4.0, 6.0) increase walking penalties, resulting in tighter, tighter clusters.

Usage: cluster mcl 2.0 10

Jaccard Filter

Mechanism: Purely topological filter. Calculates the Jaccard index (shared neighbors / total neighbors) for every edge and cuts edges falling below the threshold. Requires numba.

Parameter: Threshold (Default: 0.2). A float between 0.0 and 1.0. A threshold of 0.2 means connected nodes must share at least 20% of their surrounding neighbors to remain attached.

Usage: cluster jaccard 0.2 10

Utility Subcommands
  • cluster list: Prints a detailed report to the console detailing cluster sizes, percentages, and total noise.
Important Notes
  • Clustering is a destructive action to the visual state (overwrites node colors), but can be reverted with the undo command.
  • Clustering parameters are cached. Downstream sequence alignment tools automatically construct output directories named after your specific parameters (e.g., LEIDEN_1.0_Min10).
subcluster – Cluster Subclustering Tool
New
subcluster <cluster_name> [mode] [param_1] [min_size] | subcluster clear

Overview: Performs subclustering on a specific topology cluster, creating custom group labels named subcluster_N_M (where N is the original cluster ID, and M is the subcluster ID). Unlike main clusters, these are saved as custom group labels so nodes can keep their original cluster identities.

Arguments
  • <cluster_name>: Name of the cluster to subcluster (e.g. cluster_2, cluster_5).
  • clear: Clears all subcluster groups (subcluster_N_M) from the viewer session.
  • [min_size]: (Optional) Minimum size of subclusters to keep (Default: 10). Smaller groups are treated as Noise.
Modes & Parameters
  • leiden (Default): Leiden Community Detection. PARAM_1 represents Resolution (Default: 1.0).
  • mcl: Markov Clustering Algorithm. PARAM_1 represents Inflation (Default: 2.0).
  • jaccard: Topology Jaccard filtering. PARAM_1 represents Threshold (Default: 0.2).
Examples
  • subcluster cluster_2
  • subcluster cluster_2 mcl 2.0 5
  • subcluster cluster_5 leiden 1.5
  • subcluster clear
color – Advanced Node Styling Tool
color [expression] [color] [scale] [shape] [<expression_2> ...]

Overview: The color command allows you to rapidly alter the visual state of the network nodes (color, size, and shape) without needing to dig into complex UI menus. It utilizes the Expression Interpreter to pinpoint specific nodes.

Attributes
  • Color: Standard color names (red, blue, green, etc.) or Hex codes (#ff0000).
  • Scale: Size multiplier prefixed with x (e.g. x2 for double size, x0.5 for half size, x0 to hide).
  • Shape: circle, square, triangle, star, diamond, cross, x, vbar (vertical bar), and hbar (horizontal bar).
Usage Mechanics
  • Implicit UI Targeting (Quick Route): If nodes are selected in the viewer with the mouse, you do not need to write an expression. Simply typing color red x2 will apply styling directly to your selection.
  • Chaining Commands: You can style multiple subsets in a single rapid command, e.g. color #cluster_1# red #cluster_2# blue x1.5.
  • Attribute Stacking: You do not need to provide all three attributes. If you only want to scale, simply pass the scale, e.g., color "Mutant" x3.
  • Undo Support: Revert color modifications instantly by typing undo.
Examples
  • color red x2 triangle
  • color P106 red
  • color #cluster_1# red #noise# x0
group – Custom Group Curation Tool
group [expression] <group_name> [<expr_2> <name_2> ...] | group list | group remove <name_1>

Overview: While the cluster command relies on mathematics to divide the network into distinct communities, the group command allows for human-driven, manual curation. It allows you to tag specific nodes with custom metadata labels.

The most important distinction between a Cluster and a Group is exclusivity. A node can only ever belong to one Cluster at a time. However, a single node can belong to unlimited Groups simultaneously (e.g., a node can be tagged as #human#, #kinase#, and #mutant# all at once).

Group Naming Rules
  • No Spaces: Group names must be continuous strings. Use underscores instead of spaces (e.g. active_site).
  • Case-Insensitive: All group names are converted to lowercase internally to prevent matching errors.
  • Reserved Words: You cannot name a group noise or cluster_[N] to prevent logic overlap.
Usage Mechanics
  • Implicit UI Targeting: Selecting nodes in the viewer with the mouse and running group [NAME] assigns that group label directly to the selection.
  • Advanced Expression Targeting: Use the Expression Interpreter to assign labels to thousands of nodes instantly. Multiple expressions and names can be chained, e.g. group P106 mutant "Human" human_proteins.
Managing Groups
  • Viewing Groups: Type group list to print a statistical table showing active custom groups, node counts, and network percentages.
  • Inspecting Nodes: Left-clicking a node displays its custom groups on the second line of the tooltip header.
  • Deleting Groups: Run group remove <Group1> [Group2] to delete custom groups from the network.
Examples
  • group active_site
  • group P106 mutant
  • group list
  • group remove active_site mutant
select – Advanced Selection Tool
select [mode] <expression> | select invert | select save <filename>

Overview: The select command bridges the gap between visual intuition and rigorous mathematical querying. While you can always select nodes by right-clicking or dragging a box with your mouse, the select tool allows you to instantly highlight thousands of specific nodes using the Common Expression Interpreter. It features powerful modifiers to combine mathematical queries with your manual mouse selections, as well as export capabilities.

* Note: This command only applies to and selects visible nodes.

Selection Modes
  • change (Default): Wipes your current selection clean and selects only nodes matching the expression.
  • add / plus / include: Keeps the current selection and adds new matches to it.
  • subtract / minus / remove: Keeps the current selection but deselects nodes matching the expression.
  • filter / keep / intersect: Drill-down filter. Keeps only currently selected nodes that match the expression, dropping all others.
  • invert: Special toggle mode (takes no expressions). Inverts the selection, highlighting all other visible nodes.
Exporting Selections (select save)

Simple tools to write output lists to the disk (saves to Cache_Files/Header_Lists/):

  • Saving Headers (.txt): If you provide a `.txt` filename (e.g. select save targets.txt), generates a text file containing the exact, full FASTA headers of selected nodes.
  • Extracting Raw Sequences (.fasta): If you provide a `.fasta` filename (e.g. select save targets.fasta), extracts raw amino acid sequences from the source sequence file and writes them to a standalone FASTA file.
Examples
  • select P106
  • select add "kinase"
  • select remove #noise#
  • select keep P106
  • select invert
  • select save selected_headers.txt
label – Differential Labeling & SDP Analysis
label [target] [global_max] [cluster_min] [global_min]

Overview: The label command performs Differential Sequence Analysis. It is designed to identify Specificity-Determining Positions (SDPs)—amino acids that are highly conserved within a specific cluster or group, but vary across the broader protein family. It exports a rich, color-coded Microsoft Excel (.xlsx) report detailing exactly which residues define your sub-families.

📋 Prerequisites

Because this tool analyzes sequence data rather than just network topology, it requires:
1. An active alignment (MSA) loaded in the viewer session.
2. A reference sequence set via the reference command, which anchors residue numbers.

Targets & Parameters
  • TARGET (Default: clusters): clusters (analyzes topology clusters and manual groups) or groups (skips clusters, analyzes only custom groups).
  • GLOBAL_MAX (gmax) [Default: 30%]: Maximum global residue frequency. Highlighted residues must occur in less than this percentage globally to be considered subset-specific.
  • CLUSTER_MIN (cmin) [Default: 95%]: Minimum consensus frequency. Highlighted residues must occur in at least this percentage inside the specific cluster.
  • GLOBAL_MIN (gmin) [Default: 95%]: Universal conservation threshold. Residues exceeding this frequency globally are highlighted as universally conserved.
Understanding the Excel Output

Output is written to Results/Cluster_Label/ and contains two worksheets:

  • Subset Stats (SDP Matrix): Lists clusters/groups in rows and alignment positions in columns. Displays conserved amino acids colored by gap-occupancy.
  • Occupancy Stats: Provides gap-occupancy percentages for every position, highlighted in green. Useful for identifying insertion/deletion variations across subfamilies.
Examples
  • label
  • label 30% 90% 95%
  • label groups cmin 90%
export – FASTA Sequence Export Tool
export [target]

Overview: The export command acts as a bridge between the visual network and your local file system. It allows you to physically isolate communities of sequences (either mathematically defined clusters or manually defined groups) and save them as independent .fasta files. This is essential for taking a massive dataset, finding a specific protein family visually, and extracting those sequences for downstream alignment, HMM generation, or structural modeling.

Targets & Extraction Modes
  • clusters (Default): Run as export or export clusters. Generates a separate .fasta file for every valid mathematical cluster (e.g. Cluster_0.fasta). Noise nodes are skipped to ensure data conservation.
  • groups / group: Run as export groups. Generates a separate .fasta file for every custom group label currently defined in the session.
  • group:<Name>: Exports only specific custom groups (e.g. export group:kinase). Multiple specific groups can be chained.
Fidelity & Routing
  • Source Fidelity: Maps visual nodes back to the original source sequence file defined in configuration, preserving full sequence data.
  • Auto-Open: On Windows, successfully completing an export will automatically open the output directory in File Explorer.
Examples
  • export
  • export groups
  • export group:human
Meta Data
meta – Node Metadata Manager Tool
New
meta | meta show <property_name> | meta download [filename] [expression]

Function: Manages node metadata by allowing users to upload Excel/CSV tables, display metadata in the HUD, or download/export metadata subsets based on logical expressions.

Subcommands & Usage
  • meta (No Arguments): Opens a file dialog explorer to select metadata files (Excel .xlsx/.xls or CSV .csv) to merge into the current session. Files must contain sequence headers in the first column, property names in the first row, and property types (number/text) in the second row.
  • meta show <property_name> / meta display <property_name>: Activates a HUD text box in the bottom-right corner of the viewer displaying the value of the specified property when a node is clicked.
  • meta show clear / meta show off: Deactivates and hides the metadata property HUD.
  • meta retrieve / download / export [filename] [expression]: Exports session metadata to an Excel or CSV file in the metadata cache directory, optionally filtered by a logical expression. If no filename is provided, opens a save file dialog.
Examples
  • meta
  • meta show Organism
  • meta show clear
  • meta download {Length>500}
  • meta export selected_meta.xlsx #cluster_1#
spectrum – Spectrum Coloring Tool
New
spectrum [expression] prop:<property_name> [scheme:<color_scheme>]

Overview: Colors visible nodes along a color gradient (spectrum) based on the values of a numerical property. Automatically calls the meta display command to display the selected property in the HUD when a node is clicked.

Arguments
  • prop:<property_name> / property:<property_name>: The target numerical property name from the loaded metadata (e.g. prop:Length).
  • scheme:<color_scheme> / color:<color_scheme>: (Optional) Matplotlib colormap name. Defaults to coolwarm.
  • [expression]: (Optional) Logical expression to select which nodes are colored. If omitted, all nodes in the network are colored.
Colormap Schemes
  • Perceptually Uniform: viridis, plasma, inferno, magma, cividis
  • Sequential: Blues, BuGn, BuPu, GnBu, Greens, Greys, Oranges, OrRd, PuBu, PuBuGn, PuRd, Purples, RdPu, Reds, YlGn, YlGnBu, YlOrBr, YlOrRd
  • Diverging: coolwarm, bwr, seismic, spectral, BrBG, PiYG, PRGn, PuOr, RdBu, RdGy, RdYlBu, RdYlGn
  • Cyclic: twilight, twilight_shifted, hsv
  • Qualitative: tab10, tab20, tab20b, tab20c, Pastel1, Pastel2, Paired, Accent, Dark2, Set1, Set2, Set3
Examples
  • spectrum prop:Length
  • spectrum property:Length color:plasma
  • spectrum #cluster_1# prop:Length scheme:coolwarm
  • spectrum {Organism=*coli*} property:Length
Graphics
print – Image Export & Snapshot Tool
print [filename] [modifiers]

Overview: The print command captures exactly what you see in the viewer and exports it as a 2D image file, ready for publication or presentation. All images are automatically routed to the Results/Saved_Images/ directory.

Modifiers
  • transparent: Removes the solid background, making the output PNG transparent.
  • full: Bypasses OpenGL camera edge-clipping. Automatically pans the camera across the network, captures high-resolution tiles, and stitches them into a massive, ultra-high-resolution PNG. Can be combined with transparency.
  • svg: Shuts down the OpenGL viewer, ports data into Matplotlib, and plots a Scalable Vector Graphic (SVG) for infinite zoom. Cannot be combined with other modifiers.
Examples
  • print my_network
  • print my_network transparent
  • print my_network full transparent
  • print my_network svg
logo – Sequence Logo Generator
logo [expression] <positions> [filename] [mode] [gap_mode] [color_scheme]

Overview: The logo command generates publication-ready sequence logos (in vector .svg or transparent .png format) to visually represent amino acid conservation across a specific subset of your network. By combining this tool with the Common Expression Interpreter, you can instantly compare the binding motifs, active sites, or structural variations of different protein sub-families directly from the viewer.

📋 Prerequisites

Requires an active MSA loaded and a reference sequence set (via reference command) to anchor position coordinate numbers.

Core Arguments
  • [POSITIONS] (Required): Comma-separated list or ranges in brackets (e.g. [10-25]).
  • FILENAME: Output file ending in .svg or .png. Saved to Results/Sequence_Logos/.
  • MODE: bits (Default - Shannon Information Content) or pcts (Percentages).
  • GAP_MODE: with_gap (Default - scales stack height by occupancy) or no_gap (ignores gaps).
  • COLOR_SCHEME: Preset color scheme. Defaults to chemistry. Presets: chemistry, classic, grays, base_pairing, colorblind_safe, charge, hydrophobicity, etc.
Examples
  • logo [10-20]
  • logo #cluster_1# [1,5] pcts no_gap
  • logo [10-20] color=charge
