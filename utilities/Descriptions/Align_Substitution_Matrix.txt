=== ALIGN SUBSTITUTION MATRIX (Align_Substitution_Matrix.py) ===

Description:
This script computes an all-vs-all sequence similarity network (SSN) using traditional substitution matrices (like BLOSUM62 or PAM250) instead of embeddings. It automates this by building a local NCBI BLAST database from the input sequence FASTA file and executing pairwise alignments using the BLASTP engine. The output statistical E-values are converted into linear -Log10(E-Value) score formats to represent edge connections in the output HDF5 network, which can be visualized in the SSN viewer.

Parameters:
- INPUT_FASTA: The input sequence file (in FASTA format) containing sequences to be aligned against each other.
- MATRIX: The amino acid substitution matrix used for traditional scoring during BLAST alignments (e.g., BLOSUM45, BLOSUM50, BLOSUM62, BLOSUM80, BLOSUM90, PAM30, PAM70, PAM250).
- NUM_THREADS: The number of CPU threads allocated for BLASTP execution and parsing.
- BATCH_SIZE: The number of parsed alignments accumulated in memory before writing to disk, protecting system RAM.
- SAFE_TEMP_DIR: The temporary directory used to cache intermediate query segments and BLAST output files during the parallel run.

=== PARSE BLAST OUTPUT (Parse_BLAST_Output.py) ===

Description:
This script parses an externally generated NCBI BLAST tabular output file (outfmt 6) and converts it into a compatible HDF5 similarity network for visualization. It processes sequence alignments by mapping headers to sequence indices, converting statistical E-values into linear -Log10(E) connectivity scores, deduplicating undirected redundant edges, and serializing the final data directly into the HDF5 network format.

Parameters:
- INPUT_BLAST_TABULAR: The path to the externally generated BLAST output file in tab-separated tabular format (containing query ID, subject ID, E-value, percent identity, etc.).
